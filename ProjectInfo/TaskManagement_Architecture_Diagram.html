05 — Design Patterns Used
Architecture Patterns
📨
CQRS
Commands for writes, Queries for reads. MediatR dispatches to dedicated handlers with pipeline behaviors for logging, validation & caching.
🔄
State Pattern
Each TaskStatus has a concrete state class defining allowed transitions. InvalidStateTransitionException enforces business rules at domain level.
🎯
Strategy Pattern
ITaskSortingStrategy with Priority, DueDate, CreatedDate, Alphabetic, and Assignee strategies. Runtime selection via TaskSortingContext.
👁️
Observer Pattern
Domain events + MediatR INotificationHandlers + SignalR Hub. Multiple subscribers react to the same event independently.
🗂️
Repository Pattern
ITaskRepository, IUserRepository abstractions over EF Core. Enables unit testing with mock repositories and future DB swap.
🌳
Composite Pattern
TaskEntity holds a recursive List<TaskEntity> subtasks. GetAllSubtasks() traverses the full hierarchy. UpdateSubtaskPriorities() propagates upward.
